This is the second full audit of RRHH AMA, an internal HR management system for the Anesthesiology Residency Program at AMA Mendoza. The application is built with Astro 5 (SSR mode), Alpine.js, Supabase, and deployed on Vercel.

Since the first audit, the critical hardcoded Supabase service role key has been fully resolved — all keys now use environment variables and .gitignore correctly excludes .env files. This was the most urgent finding from the previous report.

However, this audit uncovered 1 critical, 12 high, 16 medium, and 16 low severity findings across all domains. The most urgent issue is CSRF protection being deliberately disabled in the Astro config, leaving all 25+ POST endpoints vulnerable to cross-site request forgery.

01 / 06

Security

S-01 Critical CSRF Protection Deliberately Disabled astro.config.mjs:11

Impact: All 25+ state-changing POST endpoints are vulnerable to Cross-Site Request Forgery. An attacker can craft a malicious page that submits forms to create/delete users, modify records, or perform admin operations on behalf of an authenticated user.

security: {
  checkOrigin: false, // Astro's built-in CSRF protection is OFF
},

No CSRF tokens are used anywhere in the codebase (zero results for "csrf" across all files). All forms use plain POST without origin validation.

Remediation:

  • Set checkOrigin: true in astro.config.mjs (1-line fix)
  • This validates the Origin header on all POST/PUT/DELETE requests
  • Handle exceptions individually for webhooks if needed
S-02 High Missing Content-Security-Policy Header HTTP Response Headers

Impact: No CSP means zero defense-in-depth against XSS. The app uses inline scripts extensively (Alpine.js, Chart.js), broadening the attack surface.

Remediation: Add via vercel.json or middleware. Start with: default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; connect-src 'self' https://*.supabase.co

S-03 High Open Redirect in Auth Callback src/pages/auth/callback.astro:16

Impact: The redirectTo parameter is used in Astro.redirect() without validation. An attacker can craft /auth/callback?code=X&redirectTo=https://evil.com to redirect users to phishing sites post-authentication.

return Astro.redirect(Astro.url.searchParams.get('redirectTo') || '/');

Remediation: Validate that redirectTo starts with / and does not start with //.

S-04 High Supabase Project Ref & Pooler URL in Git-Trackable Files supabase/.temp/project-ref, pooler-url

Impact: The supabase/.temp/ directory contains the project reference ID and full database pooler connection string. The .gitignore does NOT exclude this directory.

project-ref: fganbmycldqjdzbfvist
pooler-url: postgresql://postgres.fganbmycldqjdzbfvist@aws-0-us-west-2.pooler.supabase.com:5432/postgres

Remediation: Add supabase/.temp/ to .gitignore. Run git rm --cached -r supabase/.temp/ if committed.

S-05 High Multiple High-Severity Dependency Vulnerabilities package.json

Impact: npm audit reports 5 HIGH severity vulnerabilities including path traversal in Rollup 4, ReDoS in path-to-regexp/minimatch, and prototype pollution + ReDoS in xlsx (SheetJS).

Remediation: Update dependencies. Remove unused xlsx package. Run npm update for transitive dependency patches.

S-06 High Missing Security Headers (X-Frame-Options, X-Content-Type-Options, Referrer-Policy) HTTP Response Headers

Impact: App can be embedded in iframes (clickjacking), browsers may MIME-sniff responses, and sensitive URL parameters may leak to external sites.

Remediation: Add via vercel.json: X-Frame-Options: DENY, X-Content-Type-Options: nosniff, Referrer-Policy: strict-origin-when-cross-origin, Permissions-Policy.

S-07 Medium innerHTML Usage with User Data (Potential DOM XSS) src/pages/admin/usuarios.astro:358,367

Impact: User data (nombre, email) is inserted via innerHTML without sanitization. If user names contain HTML/script tags, they execute.

info.innerHTML = '<strong>' + data.nombre + '</strong> (' + data.email + ')';

Remediation: Use textContent instead of innerHTML for user-supplied data.

S-08 Medium Email Enumeration via Public check-email Endpoint src/pages/api/auth/check-email.ts

Impact: /api/auth/check-email is public and returns { exists: true/false } for any email. Rate limited (10/min) but scriptable across IPs.

Remediation: Always return the same response regardless of email existence, or require CAPTCHA.

S-09 Medium Pervasive supabaseAdmin (service_role) Usage Without RLS Write Policies src/lib/supabase.ts:17 (67+ imports)

Impact: Nearly every endpoint uses supabaseAdmin which bypasses RLS entirely. Combined with CSRF (S-01), an attacker could read/modify all data. RLS policies exist but are SELECT-only.

Remediation: Use SSR cookie-based client for authenticated operations. Reserve supabaseAdmin for privileged ops. Add RLS write policies.

S-10 Medium Weak Password Policy supabase/config.toml:175-178

Impact: Minimum password length is 6 with no complexity requirements.

minimum_password_length = 6
password_requirements = ""

Remediation: Set minimum_password_length = 12 and password_requirements = "lower_upper_letters_digits".

S-11 Low Rate Limiter Resets on Serverless Cold Starts src/lib/rate-limit.ts:10

Remediation: Use Vercel KV or Upstash Redis for persistent rate limiting. Current implementation is acceptable as defense-in-depth.

S-12 Low No robots.txt or security.txt /robots.txt, /.well-known/security.txt (404)

Remediation: Add public/robots.txt with Disallow: /. Add public/.well-known/security.txt.

Strengths

Hardcoded Supabase keys fully resolved (previous critical finding fixed)
Strong Zod input validation on every API endpoint prevents injection
Comprehensive RBAC middleware with 9 distinct access control blocks
HTTPS enforced with 308 redirect; HSTS present with preload
No sensitive paths exposed (.env, .git, package.json all 404)
CORS restrictive by default (no Access-Control-Allow-Origin)
Audit logging on all writes with PII redaction
Cookie security on impersonation (httpOnly, secure, sameSite, 4h expiry)

Accessibility

A-01 High Email Input on Login Page Has No Accessible Label src/pages/auth/login.astro:266-272

Impact: Screen reader users cannot identify the purpose of the email input field — the primary entry point for the application.

<input type="email" id="email-input" placeholder="tu@email.com" />
<!-- No label, no aria-label, no aria-labelledby -->

Remediation: Add <label for="email-input"> or aria-label="Correo electrónico".

WCAG: 1.3.1, 4.1.2, 3.3.2

A-02 High No aria-live Regions for Dynamic Error/Success Messages Entire codebase (0 matches)

Impact: Screen reader users are never notified when error or success messages appear. Zero aria-live, role="alert", or role="status" anywhere in the codebase.

Remediation: Add role="alert" to error containers. Add aria-live="polite" for success messages.

WCAG: 4.1.3

A-03 High 26+ Tables Have No Accessible Name Multiple admin/page files

Impact: Screen reader users cannot distinguish between tables. Only DataTable component has aria-label; raw tables in 15+ pages do not.

Remediation: Add <caption> or aria-label to every table.

WCAG: 1.3.1

A-04 High Login Page Tab Pattern Lacks ARIA Attributes src/pages/auth/login.astro:245-248

Impact: Screen readers cannot understand the Google/Email tabbed interface. Active state communicated only via CSS class.

Remediation: Implement WAI-ARIA tabs pattern: role="tablist", role="tab", aria-selected, aria-controls, role="tabpanel".

WCAG: 4.1.2, 1.3.1

A-05 High Focus Indicators Removed (outline:none) Without Adequate Replacement 15+ component files

Impact: Keyboard-only users cannot see which element is focused. Component styles override globals.css with outline:none and provide only 0.1-opacity box-shadow (~1.5:1 contrast, needs 3:1).

Remediation: Remove outline:none overrides or increase box-shadow opacity to 0.4+.

WCAG: 2.4.7, 2.4.11

A-06 High Text with CSS Opacity Falls Below 4.5:1 Contrast Ratio login.astro, BaseLayout.astro

Impact: Low-vision users cannot read text below WCAG AA thresholds. .login-note opacity 0.7 = 3.35:1 (FAIL). .nav-label opacity 0.6 = 2.83:1 (FAIL). .sidebar-footer opacity 0.5 = 2.36:1 (FAIL).

Remediation: Remove opacity or use a dedicated lighter color that meets 4.5:1.

WCAG: 1.4.3

A-07 High Modal Dialog Lacks Focus Trap src/components/shared/Modal.astro

Impact: Keyboard users can Tab out of modal into background content.

Remediation: Implement focus trap or use native <dialog> element.

WCAG: 2.4.3

A-08 Medium SVG Icons Lack aria-hidden or Accessible Text BaseLayout.astro, login buttons

Remediation: Add aria-hidden="true" to decorative SVGs. Add <title> for functional SVGs.

WCAG: 1.1.1

A-09 Medium User Menu Dropdown Has No aria-expanded/aria-haspopup src/components/layout/UserMenu.astro:28

Remediation: Add :aria-expanded="open" and aria-haspopup="true".

WCAG: 4.1.2

A-10 Medium View-As-Role Select Lacks Accessible Label src/components/layout/ViewAsRoleBanner.astro:64

Remediation: Add aria-label="Ver como rol".

WCAG: 4.1.2, 1.3.1

A-11 Medium Auto-Submitting Select Elements Disorienting for Keyboard Users 20+ select elements across pages

Impact: Keyboard users selecting options with arrow keys trigger unexpected form submissions and page reloads.

Remediation: Add a "Filtrar" submit button alongside selects, or debounce submission.

WCAG: 3.2.2

A-12 Medium Heading Hierarchy Issue (h2 Before h1 in DOM) src/components/layout/BaseLayout.astro:79

Remediation: Change sidebar logo from <h2> to <p> or <span>.

WCAG: 1.3.1

A-13 Low Login SVG Logo Lacks aria-hidden src/pages/auth/login.astro:224

Remediation: Add aria-hidden="true".

A-14 Low HTML lang="es" Could Be More Specific (es-AR) login.astro:17

Remediation: Change to lang="es-AR".

A-15 Low Modal Missing aria-labelledby src/components/shared/Modal.astro:38

Remediation: Add id to h2 title and aria-labelledby to dialog.

A-16 Low Spinner Lacks prefers-reduced-motion src/pages/auth/login.astro:203

Remediation: Add @media (prefers-reduced-motion: reduce) { .spinner { animation: none; } }

P-01 High Excessive Google Fonts (3 Families, 28 Files, ~378 KB) src/components/layout/BaseLayout.astro:47

Impact: Loading DM Serif Display, DM Sans (7 weights + italic), JetBrains Mono downloads ~378 KB across 28 WOFF2 files with unused subsets (cyrillic, greek, vietnamese).

Remediation: Self-host with @fontsource using Latin-only subsets (~378 KB → ~60 KB).

P-02 High Sequential Supabase Query Waterfalls src/middleware.ts:56-73, 4 page files

Impact: Every authenticated page runs 2-3 sequential middleware queries. Pages like informes/[id].astro add 3 more. Adds 100-300ms per load.

Remediation: Wrap parallel-safe queries with Promise.all(). Consider Supabase RPC for middleware.

P-03 High Dead Dependency: xlsx (SheetJS) ~1.2 MB package.json:27

Impact: Not imported anywhere. Adds ~1.2 MB and has known CVEs.

Remediation: npm uninstall xlsx — zero risk.

P-04 Medium Inline Styles 219 KB (55 Components) Multiple .astro files

Remediation: Convert repeated patterns to Tailwind utilities. Extract shared styles into globals.css @layer components.

P-05 Medium Full SSR Without Hybrid Optimization astro.config.mjs:8

Remediation: Switch to output: 'hybrid'. Add export const prerender = true to static pages. Add prefetch: { defaultStrategy: 'hover' }.

P-06 Low No vercel.json for Static Asset Caching Project root

Remediation: Create vercel.json with immutable cache headers for /_astro/*.

P-07 Low 23 Inline SVGs in Navigation (~5 KB) BaseLayout.astro:72-207

Remediation: Use SVG sprite sheet or astro-icon. Low priority.

C-02 High listUsers() Fetches ALL Auth Users on Every User Creation src/pages/api/admin/usuarios/crear.ts:64

const authMatch = authMatches?.find(u => u.email?.toLowerCase() === email);

Remediation: Use listUsers({ filter: email }) or getUserByEmail().
