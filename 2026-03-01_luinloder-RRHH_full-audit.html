Security

S-01 Medium CSP allows unsafe-inline and unsafe-eval vercel.json:18

Location: vercel.json:18

The Content Security Policy includes 'unsafe-inline' and 'unsafe-eval' in script-src, significantly weakening XSS protections. unsafe-eval is required by Alpine.js, but unsafe-inline could be replaced with nonce-based CSP.

script-src 'self' 'unsafe-inline' 'unsafe-eval' https://cdn.jsdelivr.net

Remediation: Use @alpinejs/csp build to remove unsafe-eval need. Replace unsafe-inline with nonce-based CSP.

S-02 Medium Missing ownership check in guardar-batch endpoint app/src/pages/api/cpo/guardar-batch.ts:16

Location: app/src/pages/api/cpo/guardar-batch.ts:16

The /api/cpo/guardar-batch endpoint does not verify that a CPO user can only modify records belonging to their own professional profile. Other CPO endpoints (registrar.ts, eliminar-todo.ts, registrar-bulk.ts, autoasignar.ts) all perform this ownership verification. A CPO user who knows another professional's UUID could overwrite their monthly records.

// guardar-batch.ts -- only checks if user exists, NO ownership check
const usuario = context.locals.usuario;
if (!usuario) {
  return redirect({ error: 'No autenticado' });
}

// Compare with registrar.ts:33 which HAS the check:
if (tieneRol(usuario, 'cpo') && !tieneRol(usuario, 'admin') && usuario.profesional_cpo_id !== profesional_id) {
  return context.redirect(...);
}

Remediation: Add the same ownership guard present in other CPO endpoints.
