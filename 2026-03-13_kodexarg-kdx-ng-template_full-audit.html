01 / 10

Security

SEC-001 High JWT access and refresh tokens stored in localStorage auth.service.ts:100-101

Impact: Any XSS payload or malicious script can read both tokens. The refresh token (7-day lifetime) provides persistent account access — and cannot be revoked once stolen (see SEC-003).

localStorage.setItem('access_token', access);
localStorage.setItem('refresh_token', refresh);

Remediation: Store access token in memory (a signal). Store refresh token in an HttpOnly; Secure; SameSite=Strict cookie set by the backend. Requires a backend cookie-based refresh endpoint.

SEC-002 High No Content-Security-Policy on frontend or API

Impact: Without CSP, browsers allow inline scripts and arbitrary external sources. This maximises XSS blast radius — especially critical given localStorage token storage (SEC-001).

Neither localhost:4200 nor localhost:8000 return a Content-Security-Policy header. Django SecurityMiddleware is present but no CSP_* settings are configured.

Remediation: Add at minimum default-src 'self'; script-src 'self'; object-src 'none' via django-csp (API) and CloudFront custom headers (SPA).

SEC-003 High No JWT logout / server-side token revocation config/urls.py

Impact: auth.logout() only clears localStorage — it issues no API call. The access token (60 min) and refresh token (7 days) remain valid server-side indefinitely. A stolen token cannot be invalidated.

// auth.service.ts:92-96 — no API call, only local cleanup
logout(): void {
  this.clearTokens();
  this.router.navigate(['/login']);
}

rest_framework_simplejwt.token_blacklist is not installed. ROTATE_REFRESH_TOKENS is False.

Remediation: Install simplejwt.token_blacklist, add TokenBlacklistView at /api/auth/logout/, set ROTATE_REFRESH_TOKENS: True. Frontend must POST the refresh token before clearing storage.

SEC-004 High Production authentication uses SimpleJWT — Cognito not implemented config/settings/production.py:22-36

Impact: production.py explicitly states the Cognito integration is a TODO and that the current fallback "will fail in prod if used." The production stack uses HS256 with a local secret key, not Cognito's RS256 asymmetric validation.

# production.py:36
# For now, fallback to SimpleJWT (local dev only; will fail in prod if used).
'rest_framework_simplejwt.authentication.JWTAuthentication'

Remediation: Implement Cognito JWKS validation (python-jose against the Cognito JWKS endpoint) before any production deployment. This is a pre-launch blocker.

SEC-005 Medium ALLOWED_HOSTS defaults to wildcard * in docker-compose docker-compose.yml:39

ALLOWED_HOSTS: ${ALLOWED_HOSTS:-*} — if the env var is unset (e.g., in CI), Django accepts requests for any hostname, defeating host-header validation. Combined with DEBUG=True in local.py, this exposes full stack traces.

Remediation: Change default to localhost,127.0.0.1 or use :?ALLOWED_HOSTS must be set to fail-fast.

SEC-006 Medium Hardcoded django-insecure fallback SECRET_KEY in version control config/settings/local.py:14-15

'django-insecure-dev-key-change-in-production' is committed to the repo. Anyone who clones it knows this fallback — used for signing sessions, CSRF tokens, and password reset links.

Remediation: Remove the fallback entirely. Raise ImproperlyConfigured if DJANGO_SECRET_KEY is unset.

SEC-007 Medium Brute-force login protection rate is permissive (10/hour, IP-only) config/settings/base.py:99

10 attempts/hour per IP is confirmed working (triggered HTTP 429 at attempt 6 during audit). However, distributed IPs bypass it entirely and there is no per-account lockout. The throttle also blocked cross-agent QA testing for the audit session.

Remediation: Add account-level lockout after N failures. Consider CAPTCHA for repeated failures. Disable throttle in local settings for development.

SEC-008 Medium Angular frontend serves zero security headers localhost:4200

The SPA delivery has no X-Frame-Options, X-Content-Type-Options, Referrer-Policy, or Permissions-Policy. The Django API correctly sets these via SecurityMiddleware, but the frontend CDN does not.

Remediation: Configure AWS Amplify/CloudFront custom response headers for the SPA distribution.

SEC-009 Low No token refresh before logout on 401 — users immediately kicked out on expiry error.interceptor.ts:10-13

The error interceptor calls auth.logout() directly on any 401. AuthService.refreshToken() exists but is never called from the interceptor. A 60-minute access token expiry during an active session forces re-login.

SEC-010 Low Django admin at default /admin/ path with no IP restriction config/urls.py:6

The admin panel is accessible at the well-known /admin/ path with no IP allowlist or 2FA.

Remediation: Change URL to a non-predictable path. Restrict to internal IPs at the ALB level in production.

SEC-011 Low is_staff exposed in /api/auth/me/ response apps/users/serializers.py:28

Privilege levels in responses help attackers target high-value accounts. Remove is_staff from the serializer unless the frontend requires it for UI logic.

SEC-012 Low Server header discloses uvicorn version

All API responses include server: uvicorn. Strip or replace via reverse proxy / CloudFront.

Accessibility

A11Y-001 High Icon-only logout button has no accessible name shell.component.ts:19-24

WCAG 4.1.2 — <p-button icon="pi pi-sign-out" /> with no label, aria-label, or aria-labelledby. Screen readers announce it as an unnamed button.

Fix: <p-button icon="pi pi-sign-out" aria-label="Log out" />

A11Y-002 High Form validation errors not associated with inputs via aria-describedby login.component.ts:31-43

WCAG 3.3.1 / 1.3.1 — Email and password inputs show <p-message> errors conditionally but have no aria-describedby linking to the error element, and no aria-invalid state toggling. Screen reader users are not informed of validation failures.

Fix: Add [aria-describedby]="'email-error'" and [aria-invalid]="field.touched() && field.invalid()" to each input; add matching id to error messages.

A11Y-003 Medium No skip navigation / bypass blocks mechanism shell.component.ts

WCAG 2.4.1 — Navigation renders before <main> with no skip link. Keyboard users must tab through the entire nav on every page.

A11Y-004 Medium SPA route changes not announced to screen readers app.ts

WCAG 4.1.3 — Bare <router-outlet /> with no focus management or LiveAnnouncer call on navigation. Screen reader users get no notification when the page content changes.

Fix: Implement @angular/cdk/a11y LiveAnnouncer or router event listener to announce page title on navigation.

A11Y-005 Medium Global login error has no live region — not announced on submission login.component.ts:25

WCAG 4.1.3 — The conditional <p-message> block for API errors lacks role="alert" or an aria-live wrapper. Screen readers do not announce it when it appears.

Fix: Wrap in <div role="alert" aria-live="assertive">.

A11Y-006 Medium No :focus-visible styles in global stylesheet styles.scss

WCAG 2.4.7 / 2.4.11 — No focus ring styles defined. Keyboard navigation has no visible focus indicator beyond the browser default (which PrimeNG may suppress).

A11Y-007 Medium Link color is the sole visual differentiator from surrounding text styles.scss:13-15

WCAG 1.4.1 — text-decoration: none on links with no border or other non-color indicator. Color-blind users cannot distinguish links from regular text.

A11Y-008 Low Navigation landmark missing aria-label shell.component.ts:13

WCAG 2.4.6 — <p-menubar> renders a <nav> without ariaLabel. Add ariaLabel="Main navigation".

A11Y-009 Low Dashboard renders user data as <p>/<strong> instead of <dl>/<dt>/<dd> dashboard.component.ts:13-17

WCAG 1.3.1 — Key–value data should use definition list semantics for screen readers.

Login form uses correct autocomplete attributes (email, current-password)
Submit button disabled when form is invalid — prevents empty submission
All routes define title: — document.title updates on navigation

PERF-001 High No HTTP compression on frontend or API

Neither the Angular dev server nor the Django/Uvicorn API respond to Accept-Encoding: gzip, br. No Content-Encoding header is returned. GZipMiddleware absent from Django middleware stack. For the 700 KB JS bundle, brotli would reduce transfer size to ~100–150 KB.

Remediation: Add django.middleware.gzip.GZipMiddleware to Django MIDDLEWARE (before CommonMiddleware). Enable brotli/gzip at CloudFront for the SPA.

PERF-002 Medium No database connection pooling — new TCP connection per request config/settings/base.py:58-67

DATABASES['default'] has no CONN_MAX_AGE setting. Each request opens and closes a PostgreSQL TCP connection. Under concurrent load this is a significant bottleneck.

Remediation: Add 'CONN_MAX_AGE': 60, 'CONN_HEALTH_CHECKS': True to the database config. For production on AWS App Runner, consider RDS Proxy.

PERF-003 Medium No caching layer configured

No CACHES setting anywhere. Django defaults to per-process LocMemCache. The /api/auth/me/ endpoint hits the DB on every call. The throttle backend resets on server restart.

Remediation: Add Redis (django-redis) for production. Configure throttle backend to use Redis for persistence across restarts.

PERF-004 Medium PrimeNG Aura theme (124 KB) loaded eagerly at bootstrap app.config.ts

import Aura from '@primeng/themes/aura' is a synchronous top-level import loaded before any route resolves. In production, CloudFront brotli compression mitigates this (~15-20 KB compressed). Acceptable trade-off for global theming — no practical lazy-load alternative exists for PrimeNG presets.

Strengths

All components use OnPush — no unnecessary re-renders
All routes use loadComponent() — full lazy loading
Bundle budgets configured: 500 KB warning, 1 MB error
TTFB: 2.5 ms (frontend), 5.5 ms (API) — excellent local baseline

SEO

SEO-001 High Pure CSR — crawler sees empty <app-root></app-root> angular.json (no SSR target)

No SSR or prerender configuration exists in angular.json. Crawlers fetching without JS execution receive no text content. For B2B SaaS the public login page is an indexable surface — brand name, product description, and social sharing all apply to it.

Remediation: For the login/landing page, add Angular SSR (ng add @angular/ssr) or static prerendering. Dashboard routes do not need SSR.

SEO-002 High No robots.txt — authenticated routes potentially indexable

GET /robots.txt returns 404. Without a disallow rule for /dashboard, if a crawler ever follows a shared link while authenticated, private content could be indexed.

Remediation: Create frontend/public/robots.txt with Disallow: /dashboard and Allow: /login.

SEO-003 Medium Missing <meta name="description"> src/index.html

No meta description tag exists. Google auto-generates snippets from content — for a CSR app with empty <app-root>, the SERP snippet will be blank.

SEO-004 Medium No Open Graph tags — no social sharing preview

All five OG tags absent. Shared links on LinkedIn/Slack render with no preview card.

SEO-005 Medium Generic placeholder <title> — kdx-ng-template src/index.html

15 characters (optimal: 30-60). Per-route titles ("Login", "Dashboard") are also thin. Use a TitleStrategy with brand prefix.

Strengths

<html lang="en"> set correctly
Viewport meta correctly configured
All routes define title: strings — Angular updates document.title on navigation

Functionality

FUNC-001 High Auth guard bypassed by any non-null localStorage token value auth.guard.ts:9

Confirmed via browser testing: setting localStorage.setItem('access_token', 'invalid.token.here') then navigating to /dashboard renders the full protected page. The guard checks !!this._accessToken() (signal from localStorage) and returns true immediately without any API validation.

// auth.guard.ts:9 — bypass with any string
if (auth.isAuthenticated()) return true; // ← never validates token

Remediation: Remove the early-return fast-path. Always call await auth.checkAuth(), which validates against /api/auth/me/.

FUNC-002 Medium No noAuthGuard on /login — authenticated users can access login page app.routes.ts:8-11

Confirmed: injecting valid tokens and navigating to http://localhost:4200/login renders the login form. No redirect to /dashboard occurs. Add a canActivate: [noAuthGuard] functional guard to the login route.

FUNC-003 Medium User profile not loaded on session restore — name absent from navbar auth.guard.ts:9 / auth.service.ts:72-90

When the app boots with a token in localStorage, the guard's fast-path (if (auth.isAuthenticated()) return true) returns before checkAuth() — which calls loadProfile() — is ever invoked. The Shell's @if (auth.currentUser(); as user) block renders nothing. User name is absent from the navbar.

Fix: Always call await auth.checkAuth() in the guard (removes fast-path), or call auth.loadProfile() via APP_INITIALIZER when a token is found.

FUNC-004 Medium Raw API throttle error message exposed verbatim to users login.component.ts:108-113

When rate-limited, users see: "Request was throttled. Expected available in 3551 seconds." — a backend technical message. The component renders err.error?.detail without checking the HTTP status code first.

Fix: Detect HTTP 429 and show: "Too many login attempts. Please try again in a few minutes."

Framework Best Practices — Angular 21 + Django 5.2

FW-001 High All HTTP calls use firstValueFrom() instead of httpResource() / resource() auth.service.ts:39, 47, 58

The project's own CLAUDE.md explicitly lists firstValueFrom() as a prohibited pattern. httpResource() is the mandated Angular 21 approach for data fetching. Zero usage of either httpResource() or resource() anywhere in the codebase.

// auth.service.ts — three violations
await firstValueFrom(this.http.post<AuthTokens>('/api/auth/login/', ...));
await firstValueFrom(this.http.get<UserProfile>('/api/auth/me/'));
await firstValueFrom(this.http.post<{ access: string }>('/api/auth/token/refresh/', ...));

Remediation: Migrate loadProfile() to httpResource(). For login/refresh mutations, use resource() with a request signal or document the exception explicitly.

FW-002 High Production uses HS256 SimpleJWT — Cognito integration incomplete

See SEC-004 / OPS-002. Code explicitly states it "will fail in prod if used." Pre-launch blocker.

FW-003 Medium TokenRefreshView has no throttle — brute-forceable at 100/hour config/urls.py:7

The TokenRefreshView is registered directly from simplejwt with no custom throttle class. Global AnonRateThrottle applies at 100/hour — far too permissive for a refresh token endpoint.

Remediation: Subclass TokenRefreshView and add a tight AnonRateThrottle scope (e.g., 'token_refresh': '20/hour').

FW-004 Medium No custom ErrorHandler — bootstrap errors swallowed to console.error main.ts:5-6

bootstrapApplication(...).catch((err) => console.error(err)). provideBrowserGlobalErrorListeners() is present but no custom ErrorHandler is registered. Unhandled runtime errors are not reported to any monitoring service.

FW-005 Medium MeView throttle is implicit (global default only) apps/users/views.py:31

LoginView explicitly declares throttle_classes = [LoginThrottle] — good pattern. MeView has no throttle_classes and relies only on global 1000/hour default. Explicit throttling makes the contract visible and auditable.
