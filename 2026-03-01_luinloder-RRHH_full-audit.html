01 / 10

Security

S-01 Medium CSP allows unsafe-inline and unsafe-eval in script-src vercel.json:18

Impact: Weakens XSS protection. unsafe-eval allows eval() and Function() dynamic code execution.

script-src 'self' 'unsafe-inline' 'unsafe-eval' https://cdn.jsdelivr.net

Remediation: Use nonce-based CSP for scripts. Use Alpine.js CSP build (alpine.csp.js) to remove unsafe-eval. Remove cdn.jsdelivr.net (unused).

S-02 Medium supabaseAdmin falls back silently to anon key in non-production lib/supabase.ts:21

Impact: In dev/staging, admin operations go through RLS, masking authorization bugs that only surface in production.

export const supabaseAdmin = createClient(supabaseUrl, supabaseServiceKey || supabaseAnonKey);

Remediation: Throw an error in all environments when the service role key is missing.

S-08 Medium Rate limiting is in-memory only (resets on cold starts) lib/rate-limit.ts:1-10

Impact: Can be bypassed by waiting for Vercel function recycling or targeting different instances. Only 3 routes are rate-limited.

Remediation: Use Vercel WAF rate limiting or Upstash Redis for persistent rate limiting.

S-03 Low Unused client-side Supabase export in server module lib/supabase.ts:18

The anon-key supabase export is created but only supabaseAdmin is imported server-side. Remove or move to a client-only file.

S-04 Low Missing Cross-Origin-Embedder-Policy and Cross-Origin-Resource-Policy headers vercel.json

Two cross-origin isolation headers absent. 8/10 security headers present. Grade: B.

S-07 Low x-html directive in AlpineJS (potential DOM-based XSS vector) residencias/simulador.astro:408

Uses x-html to render computed HTML. Currently safe (numeric values only) but fragile to future changes. Prefer x-text.

S-09 Low Email enumeration via /api/auth/check-email endpoint api/auth/check-email.ts:33

Returns { exists: true/false } for any email. Rate-limited (10 req/60s) but allows slow enumeration.

S-14 Low User deletion cascade is not transactional api/admin/usuarios/eliminar.ts:40-116

8+ sequential Supabase operations without a DB transaction. Partial failure leaves inconsistent state. Use a Supabase RPC function.

Strengths

Comprehensive security headers (HSTS with preload, CSP, X-Frame-Options DENY, COOP)
CSRF protection via Astro's security.checkOrigin: true
Uses supabase.auth.getUser() (server-validated) instead of getSession()
Role-based access control with impersonation guard (rolReal)
All API routes use Zod schemas via parseFormData()
No hardcoded secrets in source code
Open redirect protection in auth callback and referer-based redirects
Audit logging with before/after data on all destructive operations
Cookie security: httpOnly, secure, sameSite, 4hr expiry
Sentry PII scrubbing (email and IP removed from error reports)

02 / 10

Accessibility

A-01 Critical 97 decorative SVG icons missing aria-hidden="true" 25 files across src/

WCAG 1.1.1 Non-text Content (Level A). Screen readers attempt to parse SVG path data, creating confusing output. All are decorative icons next to text labels.

Remediation: Add aria-hidden="true" to all decorative SVGs. Single regex find/replace operation.

A-02 High Duplicate h1 headings on multiple pages BaseLayout.astro:224 + 9 pages

WCAG 1.3.1 (Level A). BaseLayout renders <h1> in header; several pages add a second <h1> in content area.

Remediation: Change in-page <h1> to <h2>, or change BaseLayout header to a styled <div>.

A-03 High outline: none removes focus indicator on 20+ elements 20 files (login, admin, CPO pages)

WCAG 2.4.7 Focus Visible (Level AA). Despite a global :focus-visible rule with !important, many page-level CSS rules set outline: none on :focus, overriding for assistive tech that triggers :focus but not :focus-visible.

Remediation: Replace :focus { outline: none } with :focus-visible or remove them entirely.

A-04 High User menu button missing aria-expanded and aria-haspopup UserMenu.astro:28

WCAG 4.1.2 (Level A). Screen reader users cannot know the button opens a dropdown or its current state.

Remediation: Add aria-haspopup="true" :aria-expanded="open" aria-label="User menu".

A-05 High No aria-live regions for dynamic content / error messages login.astro, registro.astro, multiple

WCAG 4.1.3 Status Messages (Level AA). Zero aria-live or role="alert" found. Login errors, form success/error callouts, and Alpine reactive updates are invisible to screen readers.

Remediation: Add role="alert" to error containers and aria-live="polite" to success messages.

A-06 Medium Empty alt="" on avatar images in admin panel admin/index.astro:163

WCAG 1.1.1. Avatars treated as decorative; acceptable when name is adjacent.

A-07 Medium Missing footer landmark in BaseLayout BaseLayout.astro:212-215

WCAG 1.3.1. Sidebar footer uses <div> instead of <footer>.

A-08 Medium Sidebar navigation missing aria-label BaseLayout.astro:85

WCAG 1.3.1. Two <nav> elements without distinguishing labels. Add aria-label="Main menu".

A-09 Medium Cookie banner not accessible as a landmark/dialog CookieBanner.astro, login.astro:432

WCAG 2.4.1. Lacks role="dialog", aria-label, and focus trapping.

A-10 Medium Aside sidebar missing aria-label BaseLayout.astro:68

WCAG 4.1.2. The <aside> landmark needs a label to distinguish it.

A-11 Low Mobile menu does not trap focus BaseLayout.astro:274-288

WCAG 2.4.3. Focus escapes sidebar overlay on mobile. Use AlpineJS x-trap (already a dependency).

A-12 Low Color contrast concerns in dark theme with opacity modifiers globals.css, DataTable.astro:80

WCAG 1.4.3. .table-count uses opacity: 0.6 on muted text, reducing contrast below AA.

A-13 Low onchange auto-submit on select elements admin/usuarios.astro:82, 9 other pages

WCAG 3.2.2 On Input. Filter selects auto-submit, which is disorienting for screen reader and keyboard users.

Strengths

Skip navigation link with proper focus styling
Correct lang="es" on all pages
Login page: proper ARIA tab pattern with roles, aria-selected, aria-controls
All data tables include aria-label and scope="col"
Modal uses x-trap.noscroll, role="dialog", aria-modal, escape key
Form labels properly associated via for/id
Global :focus-visible rule with !important
Canvas charts include role="img" and aria-label
Active nav state with aria-current="page"
Combobox pattern in demand assignment autocomplete with proper ARIA

P-01 Medium No HTTP cache headers for SSR pages astro.config.mjs, vercel.json

All pages run full SSR with no Cache-Control headers. Every page load triggers a full server render + Supabase queries. No prerender exports found.

Remediation: Add s-maxage and stale-while-revalidate for semi-static pages. Use prerender = true for legal pages.

P-02 Medium Heavy inline scripts on complex pages (up to 70 KB) simulador.astro (900+ lines), CalendarioDDJJ.astro (33 KB)

is:inline scripts bypass Astro's bundler — no tree-shaking, no minification, no browser caching. The simulador page alone has 70 KB of inline JS.

Remediation: Extract to external .ts files with standard <script> tags. Pass data via data-* attributes.

P-03 Medium Six font files loaded on every page (~200-300 KB) globals.css:2-7

All 6 font weight/family files load on every page even when only a subset is used. Login page duplicates 5 imports.

Remediation: Reduce to 2-3 weights. Use font-display: swap. Subset for Latin characters.

P-07 Medium Heavy data pages with 7-9 parallel Supabase queries cpo/dashboard.astro:78, cpo/registro.astro:70

CPO dashboard executes 7 parallel queries; registro does 9. Good use of Promise.all() but high TTFB impact.

Remediation: Create Supabase DB views or RPC functions to aggregate data server-side.

P-04 Low No prerendering for static legal pages

Add export const prerender = true to privacidad.astro and terminos.astro.

P-06 Low Middleware executes 2-3 Supabase queries per authenticated request middleware.ts:66-94

Auth check + user profile + optional email fallback. Consider caching user profile in encrypted session cookie.

P-10 Low CSS animations are no-ops; stagger classes undefined globals.css:536-552

fadeInUp and slideInLeft both just fade opacity with no positional transform. stagger-* classes have no CSS rules.

Strengths

Excellent Supabase query parallelization with Promise.all()
Self-hosted fonts via @fontsource (no CDN dependency)
Chart.js lazy loaded via singleton dynamic import()
Zero raster images — SVG-only UI
No external CDN dependencies
Heartbeat write is throttled (60s) and fire-and-forget
Sentry server-side only with 10% sampling
Tailwind purge properly configured

Functionality

FN-01 High Vercel Attack Challenge Mode blocks all non-browser clients Vercel dashboard config

Impact: Every HTTP request returns 403 with x-vercel-mitigated: challenge. Health checks, monitoring tools, CI pipelines, and search crawlers all blocked.

Remediation: Exclude /api/health from Attack Challenge Mode via Vercel Firewall Rules. Evaluate if the WAF needs to be this aggressive for an internal app.

FN-02 High User deletion cascade is not atomic (no transaction) api/admin/usuarios/eliminar.ts:38-116

8+ sequential Supabase operations without a DB transaction. Partial failure = orphaned/inconsistent records.

Remediation: Use a Supabase RPC stored procedure with BEGIN...COMMIT, or use database-level ON DELETE CASCADE constraints.

FN-04 Medium API endpoints use POST redirects with error messages in query params All api/ endpoints, admin-utils.ts:58-76

PRG pattern works but error messages in URLs are visible, bookmarkable, and logged. No JSON API for richer interactions.

FN-05 Medium Rate limiter resets on cold starts (in-memory store) lib/rate-limit.ts:10

Code comment acknowledges: "Best-effort: resets on cold starts." Acceptable for low-traffic internal app but should be documented as accepted risk.

FN-06 Medium Heartbeat throttle uses unbounded in-memory Map middleware.ts:15

No cleanup logic. Benign on serverless but leaks on long-running servers.

FN-07 Medium innerHTML usage in error handler (minor XSS vector) GraficoDemanda.astro:189

Currently injects hardcoded string (safe) but the pattern is risky for future changes. Use textContent + DOM API.

FN-09 Low Detailed Supabase error messages exposed in URL query parameters Multiple api/ endpoints

May leak DB schema details. Map errors to user-friendly messages.

FN-10 Low Hardcoded KPI description "4 cohortes / 2 sedes" index.astro:76

Becomes misleading if cohort/site count changes. Compute from query results.

FN-11 Low 404 page requires auth (uses BaseLayout) 404.astro

Unauthenticated users hitting invalid URLs get redirected to login instead of seeing a 404.

FN-12 Low CPO write route role mismatch between middleware and endpoint middleware.ts:209 vs autoasignar.ts:15

Middleware allows ['cpo', 'admin'] but endpoint also allows 'coordinador'. The middleware blocks first, so coordinador is silently denied.

Dependency Health

D-01 High path-to-regexp ReDoS vulnerability via @astrojs/vercel

@astrojs/vercel@^9.0.4 depends on @vercel/routing-utils which pulls in vulnerable path-to-regexp versions (4.0.0-6.2.2).

Remediation: Monitor for patched @astrojs/vercel@9.x. Consider npm overrides for @vercel/routing-utils.

D-03 High Multiple lock files: bun.lock AND package-lock.json coexist app/bun.lock + app/package-lock.json

Two lock files = non-deterministic installs depending on which package manager runs. package-lock.json is 2 days older.

Remediation: Choose bun, delete package-lock.json, add packageManager field to package.json.

Transitive vulnerability through yaml-language-server. Dev-only, no runtime risk.

D-07 Low Zod v3 in use, v4 available

Strengths
