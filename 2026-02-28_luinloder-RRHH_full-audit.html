01 / 06

Security

S-01 High CSP allows unsafe-inline and unsafe-eval for scripts vercel.json:17

Impact: unsafe-inline completely undermines CSP XSS protection. unsafe-eval enables eval()-based attacks. Combined with innerHTML usage in the codebase, this is a real risk amplifier.

script-src 'self' 'unsafe-inline' 'unsafe-eval' https://cdn.jsdelivr.net

Remediation: Migrate to nonce-based CSP. Alpine.js v3.15+ supports CSP mode without unsafe-eval.

S-02 Medium DOM-based XSS via innerHTML with user-controlled data admin/usuarios.astro:358

Impact: If an admin sets a user name containing HTML/JS, it executes in other admins' browsers via innerHTML. Mitigated by Zod validation and admin-only access.

info.innerHTML = '<strong>' + data.nombre + '</strong>'

Remediation: Use textContent instead of innerHTML.

S-03 Medium Multiple tables missing RLS policies supabase/migrations/

Impact: Tables informes_ejecutivos, informe_acceso, and asignaciones_cpo_mensuales lack RLS. Authenticated users could read them via the anon key.

Remediation: Enable RLS on all tables and add appropriate SELECT/INSERT/UPDATE/DELETE policies.

S-04 Medium Signout endpoint uses GET method (CSRF via links/images) api/auth/signout.ts:4

Impact: A malicious site can sign out users via <img src=".../api/auth/signout">. Astro's checkOrigin only applies to POST.

Remediation: Change to export const POST and update frontend to use a form with method POST.

S-05 Medium supabaseAdmin falls back silently to anon key lib/supabase.ts:17

Impact: If SUPABASE_SERVICE_ROLE_KEY is unset, admin operations silently run under RLS. Could cause data access failures or silent permission bypasses.

export const supabaseAdmin = createClient(supabaseUrl, supabaseServiceKey || supabaseAnonKey);

Remediation: Throw an error at startup if the service key is missing in production.

S-06 Medium auth.admin.listUsers() called without pagination api/admin/usuarios/crear.ts:64

Impact: Downloads entire auth.users table into memory on every user creation. O(N) scan, fails silently for users beyond first page (1000 default).

Remediation: Use targeted lookup by email instead of listing all users.

S-07 Low Email enumeration via /api/auth/check-email api/auth/check-email.ts:33

Impact: Rate limited to 10 req/60s, but still allows enumeration over time. Leaks which emails are registered.

Remediation: Consider always returning {exists: true} and handling non-registered users at callback stage.

S-08 Low In-memory rate limiter resets on Vercel cold starts lib/rate-limit.ts:10

Impact: Each cold start creates a fresh rate limit store. Attackers can bypass by waiting for function recycling.

Remediation: Use persistent rate limiter (Upstash Redis or Vercel KV) for critical endpoints.

S-09 Low Missing Cross-Origin-Opener-Policy header All routes

Impact: Without COOP, the app may be vulnerable to cross-origin window manipulation (Spectre-class attacks).

Remediation: Add Cross-Origin-Opener-Policy: same-origin to vercel.json headers.

S-10 Low Dependency vulnerabilities (5 high, 2 moderate, 2 low) package.json (transitive)

Impact: Transitive vulnerabilities in minimatch (ReDoS), rollup (path traversal), path-to-regexp (ReDoS). Mostly build/dev tooling, not runtime.

Remediation: Run bun update to pull latest compatible versions. Add overrides for remaining transitive issues.

02 / 06

Accessibility

A-01 High Multiple h1 elements on the homepage index.astro:60-61

Impact: Three h1 elements on the homepage violate WCAG 1.3.1. Screen readers receive conflicting signals about the page's primary topic.

Remediation: Use a single h1 for the hero section. Change BaseLayout header to h2 or demote hero headings.

A-02 High outline:none removes focus indicators on 25+ form elements Multiple files

Impact: Keyboard users cannot see which element has focus. The global :focus-visible rule is overridden by page-level outline:none with only a subtle 1px border change that fails the 3:1 contrast requirement (WCAG 2.4.7/2.4.11).

Remediation: Remove page-level outline:none overrides or add box-shadow: 0 0 0 2px rgba(78, 205, 196, 0.3) alongside every instance.

A-03 High Data tables lack scope attributes and caption elements All table pages

Impact: Screen readers cannot determine header-to-data cell relationships. Zero uses of scope= or <caption> across the entire codebase. Violates WCAG 1.3.1.

Remediation: Add scope="col" to all <th> elements and aria-label or <caption> on each table.

A-04 Medium ViewAsRoleBanner select has no associated label ViewAsRoleBanner.astro:64

Impact: Screen readers announce "combobox" without context. Placeholder text is not programmatically associated.

Remediation: Add aria-label="Ver como otro rol" to the select element.

A-05 Medium Profesional-select dropdown in registro.astro lacks label cpo/registro.astro:249

Impact: Screen readers cannot identify the dropdown purpose. Violates WCAG 4.1.2.

Remediation: Add aria-label="Seleccionar profesional".

A-06 Medium Sidebar logo h2 creates heading level skip before h1 BaseLayout.astro:81

Impact: h2 appears before h1 in DOM order, confusing screen reader heading navigation.

Remediation: Change sidebar logo from <h2> to <p> with appropriate styling.

A-07 Medium Login page lacks landmark regions auth/login.astro

Impact: No <main>, <header>, or <nav> on the standalone login page. Violates WCAG 1.3.1.

Remediation: Wrap the login card in a <main> element.

A-08 Medium Login mode tabs missing ARIA tab pattern auth/login.astro:247

Impact: Google/Email tabs appear as generic buttons. No role="tablist", role="tab", aria-selected, or aria-controls.

Remediation: Implement the full ARIA tabs pattern with tablist, tab, and tabpanel roles.

A-09 Medium Delete confirmation input not connected to label admin/usuarios.astro:297

Impact: Screen readers don't communicate why the delete button is disabled or what input enables it.

Remediation: Add for= on the label and id on the input, plus aria-describedby.

A-10 Medium Autocomplete combobox in demanda.astro lacks ARIA attributes cpo/demanda.astro:636

Impact: The inline table autocomplete is missing role="combobox" and related attributes. The ResumenMesDemanda component does this correctly.

Remediation: Add ARIA combobox attributes matching the pattern in ResumenMesDemanda.astro.

A-11 Medium Admin form labels duplicated across all CRUD forms AdminFormField.astro

Impact: Screen readers announce "Email Email", "Nombre Nombre", "Tipo Tipo Tipo" across admin pages. Systematic pattern from both visible label and aria-label/placeholder being used.

Remediation: Use either a visible <label> OR aria-label, not both with the same text.

A-12 Low Sidebar navigation does not indicate current page BaseLayout.astro:260

Impact: No aria-current="page" on the active link. Visual-only indication via accent color.

Remediation: Add link.setAttribute('aria-current', 'page') when active.

A-13 Low Mobile hamburger menu lacks focus trap BaseLayout.astro:259

Impact: Tab key can escape the open sidebar into underlying content. Focus not returned to toggle on close.

Remediation: Implement focus trapping when sidebar is open, return focus on close.

A-14 Low Cookie banner buttons lack focus styles CookieBanner.astro:26

Impact: Inline styles override global :focus-visible. Low impact since banner appears once.

Remediation: Move to class-based styling or add inline focus ring.

A-15 Low Delete buttons use single 'x' character as accessible name cpo/licencias.astro:311

Impact: Screen readers announce "x, button" which is not meaningful. title="Eliminar" is not consistently announced.

Remediation: Add aria-label="Eliminar licencia" to the button.

P-01 Critical CPO Registro page HTML is 2.5 MB decoded cpo/registro.astro

Impact: Browser must parse and render a 2.5 MB DOM. Causes 1.8s FCP and 2.8s DOM Complete. Smoke test confirmed 295ms TTFB (highest of all pages) and 1083ms DOM Complete even with warm cache.

Remediation: Paginate by hospital or lazy-render calendar sections. Consider client-side rendering for calendar grids fetched via API.

P-02 High CPO Dashboard page is 400 KB decoded with 2.5s FCP cpo/dashboard.astro

Impact: FCP exceeds the 1.8s "good" threshold. 7 parallel DB queries plus complex client-side grouping. Chart.js adds 74 KB.

Remediation: Use Supabase views/RPCs for server-side aggregation. Lazy-load Chart.js when chart section enters viewport.

P-03 High Google Fonts loads 11 variants render-blocking BaseLayout.astro:49

Impact: DM Serif Display, DM Sans (7 weights), JetBrains Mono (2 weights) block rendering by 200-400ms. Many weights remain unloaded.

Remediation: Self-host fonts via @fontsource. Reduce to only needed weights. Use link rel="preload" for critical fonts.

P-04 High Login page loads unique font families not used elsewhere auth/login.astro:28

Impact: Literata and Playfair Display loaded only on login, differing from the rest of the app's DM Serif Display.

Remediation: Replace with DM Serif Display for consistency. Remove unused font families.

P-05 High site.webmanifest fetched twice per page load (68 KB wasted) BaseLayout.astro:45

Impact: 34 KB manifest fetched twice on every page navigation. The manifest is only 269 bytes on disk, so the 34 KB response may be Vercel's protection page.

Remediation: Investigate duplicate manifest link tags. Verify the manifest is served correctly through Vercel protection.

P-06 Medium Excessive select('*') causes over-fetching from Supabase 45+ occurrences

Impact: Transfers unnecessary columns from Supabase, increasing response payload and TTFB.

Remediation: Replace select('*') with explicit column lists.

P-07 Medium Massive inline style/script blocks prevent cross-page caching Multiple pages

Impact: Pages embed 600+ lines of CSS/JS inline. Cannot be shared or cached across navigations.

Remediation: Extract common styles into shared CSS files. Move Alpine.js logic into separate .ts files.

P-08 Medium Chart.js loaded as full bundle (74 KB) not tree-shaken lib/chart-loader.ts

Impact: All chart types loaded but only bar/line used. Could be reduced to ~30-40 KB.

Remediation: Import only needed components: BarController, BarElement, CategoryScale, LinearScale, etc.

P-09 Medium Middleware performs 2-3 sequential Supabase queries per request middleware.ts:63

Impact: Adds 50-150ms to every page load. getUser() + usuario lookup + optional fallback lookup, all sequential.

Remediation: Cache usuario profile in a signed cookie. Parallelize where possible.

P-10 Medium Rotaciones page shows elevated TTFB (623ms vs 65ms avg) mi-residencia/rotaciones

Impact: Smoke test measured 623ms TTFB — nearly 10x the average. Indicates a slow server-side query or heavy computation.

Remediation: Profile the rotaciones page data fetching. Check for missing database indexes or N+1 query patterns.

P-11 Low Admin profesionales page has slowest DOM Complete (1469ms) admin/profesionales.astro

Impact: Smoke test: 1469ms DOM Complete. Loads 47 professionals with status management and complex table rendering.

Remediation: Implement server-side pagination if professional count grows.

04 / 06

SEO

SEO-01 High Vercel Attack Challenge Mode blocks all crawlers All routes

Impact: Every unauthenticated request returns 403 with x-vercel-mitigated: challenge. Zero indexability. The site is invisible to search engines.

Remediation: Intentional for internal app. If public pages need indexing, configure Vercel WAF to exempt specific paths.

SEO-02 High No robots.txt file exists public/ directory

Impact: Crawlers have no guidance on which paths to index. For an internal app, a disallow-all robots.txt is best practice.

User-agent: *
Disallow: /
Allow: /auth/login
Allow: /legal/
SEO-03 Medium No canonical URLs on any page BaseLayout.astro:35-50

Impact: Risk of duplicate content via URL variants (trailing slash, query params).

Remediation: Add <link rel="canonical"> to BaseLayout and set site in astro.config.mjs.

SEO-04 Medium No Open Graph tags on any page BaseLayout.astro:35-50

Impact: Shared links on Slack, WhatsApp, Teams show no preview card — only a bare URL.

Remediation: Add og:title, og:description, og:type, og:url meta tags to BaseLayout.

SEO-05 Medium Identical meta description on all BaseLayout pages BaseLayout.astro:12

Impact: All 25 authenticated pages share the exact same description. Search engines cannot differentiate pages.

Remediation: Pass unique description props per page.

SEO-06 Medium Standalone pages missing meta description auth/login.astro, legal/

Impact: The only public-facing pages have no description for search engines or link previews.

Remediation: Add <meta name="description"> to login and legal pages.

SEO-07 Medium No X-Robots-Tag header for defense-in-depth vercel.json

Impact: If a crawler bypasses Vercel's challenge, pages could be indexed.

Remediation: Add X-Robots-Tag: noindex, nofollow to vercel.json headers.

SEO-08 Low No sitemap.xml exists public/ directory

Impact: No URL discovery mechanism. Acceptable for internal app if robots.txt disallows crawling.

Remediation: Not needed if robots.txt correctly disallows all.

SEO-09 Low No structured data (JSON-LD) on any page All source files

Impact: No rich snippets. Near-zero practical impact for internal app.

Remediation: Not needed for internal app.

SEO-10 Low No Twitter Card tags BaseLayout.astro

Impact: Links shared on Twitter/X have no rich preview. Low priority for internal app.

Remediation: Add twitter:card, twitter:title, twitter:description if desired.

Code Quality

Strengths

Comprehensive Zod validation on every API route with shared schema primitives
Well-structured RBAC with impersonation safety (rolReal vs rolUI separation)
Audit logging on all write operations with PII redaction via stripPII()
Safe OAuth callback redirect validation preventing open redirects

C-01 Critical supabaseAdmin falls back to anon key silently when service role key missing lib/supabase.ts:17

Impact: If SUPABASE_SERVICE_ROLE_KEY is unset, all admin operations silently run under RLS. No warning logged.

export const supabaseAdmin = createClient(supabaseUrl, supabaseServiceKey || supabaseAnonKey);

Remediation: Throw an error at startup if the key is missing: if (!supabaseServiceKey) throw new Error('SUPABASE_SERVICE_ROLE_KEY required')

C-02 Critical listUsers() fetches ALL auth users on every user creation api/admin/usuarios/crear.ts:64

Impact: Downloads entire auth.users table into memory. O(N) scan that degrades with user count. Silently fails for users beyond page 1 (1000 default).

Remediation: Use targeted lookup: supabase.auth.admin.listUsers({ filter: 'email.eq.'+email })

C-03 High Non-transactional cascade delete risks partial deletion api/admin/usuarios/eliminar.ts:38-116

Impact: 8 sequential Supabase calls to nullify FKs and delete entities with no transaction wrapping. Intermediate failure leaves orphaned data.

Remediation: Wrap in a Supabase RPC (Postgres function) for atomic execution. At minimum, check errors at each step.

C-04 High GET signout vulnerable to CSRF via links/images api/auth/signout.ts:4

Impact: State-changing operation via GET. checkOrigin only applies to POST/PUT/DELETE.

Remediation: Change to export const POST. Update frontend to use form with method POST.

C-05 High Pervasive use of 'any' type (~120+ occurrences) Multiple files

Impact: TypeScript type system disabled across most complex pages. Runtime errors from schema changes cannot be caught at compile time. The strict tsconfig is rendered ineffective.

Remediation: Parameterize Supabase clients with Database generic from existing database.types.ts. This single change eliminates most any annotations.

C-06 High Oversized page components (900-1100 lines) cpo/demanda.astro, dashboard.astro, registro.astro

Impact: Monolithic files mix data layer, business logic, presentation, and styling. Hard to test, review, and refactor.

Remediation: Extract data fetching into lib/ modules. Keep .astro files focused on template rendering.

C-07 Medium CSP allows unsafe-inline and unsafe-eval vercel.json:13

Impact: Effectively negates XSS protection from CSP. Covered in detail in Security section (S-01).

Remediation: Use nonce-based CSP. Consider Alpine.js CSP-compatible build.

C-08 Medium Audit log writes are fire-and-forget with no error handling lib/admin-utils.ts:111

Impact: Failed audit log inserts are silently swallowed. For an HR compliance system, missing audit entries could be a legal issue.

Remediation: Check insert errors and log to Sentry. Consider whether audit failure should block the operation.

C-09 Medium In-memory rate limiting resets on serverless cold starts lib/rate-limit.ts:10

Impact: Rate limiting is non-functional under moderate traffic. Provides false sense of security.

Remediation: Use a persistent store (Upstash Redis, Vercel KV). Document current approach as best-effort only.

C-10 Medium Hardcoded year '2025' in template text cpo/demanda.astro:538

Impact: Label reads "Datos Historicos 2025" which is already outdated.

Remediation: Use dynamic date: {new Date().getFullYear()}

C-11 Medium Missing usuario_id in several audit log calls api/admin/usuarios/crear.ts:82

Impact: Audit entries have null usuario_id, making it impossible to trace who performed the action.

Remediation: Add usuario_id: context.locals.usuario?.id to all writeAuditLog calls.

C-12 Low Unused client-side Supabase client exported lib/supabase.ts:14

Impact: Dead code. All imports use supabaseAdmin. Login page creates its own browser client.

Remediation: Remove the unused export.

C-13 Low @astrojs/check listed as runtime dependency package.json:19

Impact: Type-checking tool included in production dependencies unnecessarily.

Remediation: Move to devDependencies.

C-14 Low Inconsistent error messages — mixed Spanish and English Throughout API routes

Impact: Mixed languages in user-facing messages. Spanish messages lack proper accent marks.

Remediation: Standardize on Spanish for user-facing, English for code comments/JSDoc.

C-15 Low Duplicate Zod validation boilerplate in every API route 30+ API routes

Impact: High repetition. Any change to error handling requires modifying 30+ files.

Remediation: Create a parseOrRedirect helper to encapsulate the pattern.

06 / 06

Functionality

F-01 Critical Role switcher produces 'Cross-site POST form submissions are forbidden' All pages — role switcher

Impact: The role-switching feature is completely non-functional. Selecting any role produces a blank white page with the error. Users must manually navigate back. Core feature for Coordinador/Admin users.

Remediation: Ensure the form POST action matches the deployment origin. Consider using a client-side API call (fetch) instead of a form POST.

F-02 High Multiple pages return 404 when accessed via direct URL CPO section routes

Impact: /cpo/dashboard-bi, /cpo/registro-ddjj, /cpo/profesionales show 404 on direct URL but work via nav clicks. Bookmarks, refresh, and link sharing break. Note: smoke test showed all direct URL routes working — this may be intermittent or related to Vercel's routing cache.

Remediation: Verify Astro SSR route configuration and Vercel rewrites for all CPO routes.

F-03 Medium No confirmation dialog on destructive admin delete actions admin/usuarios

Impact: Single click could delete a user account. 90 users listed without pagination increases misclick risk.

Remediation: Add a confirmation dialog before executing destructive actions.

F-04 Medium Hospitales filter label displayed three times ('Tipo Tipo Tipo') admin/hospitales

Impact: Screen readers announce the label repeated. Visual duplication depending on CSS.

Remediation: Fix the admin form component to use only one label association.

F-05 Low Cookie/Privacy banner button was unresponsive during testing Simulador R/Q page

Impact: Users may not be able to dismiss the consent banner. Appeared once, did not reappear.

Remediation: Verify banner button click handlers and z-index across all pages.

F-06 Low User menu only contains sign-out option All pages — GC Gabriel button

Impact: No profile, settings, or account management. Limited user self-service.

Remediation: Consider adding profile viewing and preferences to the user menu.
