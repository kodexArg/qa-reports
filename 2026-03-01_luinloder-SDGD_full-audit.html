S-01 Critical Hardcoded database password committed to git scripts/run-sql.mjs:18

Location: scripts/run-sql.mjs:18

Full PostgreSQL credentials (host, user, password, database) for the production Supabase database are hardcoded in a tracked file. Password: plaintext string. Host: aws-1-us-east-2.pooler.supabase.com.

Impact: Anyone with repository access has direct database credentials bypassing all application security, RLS, and authentication.

Remediation:

  • Rotate the database password immediately in Supabase dashboard
  • Remove the file from git history using BFG Repo Cleaner
  • Use environment variables instead

Location: .claude/settings.local.json:4

The Supabase management access token (sbp_2e010d1d...) is committed to git. This token allows full control over the Supabase project: database management, API key rotation, project settings.

Impact: Complete Supabase project compromise. Any person with repo access can manage the entire backend.

Remediation:

  • Revoke the token in Supabase dashboard (Account > Access Tokens)
  • Add .claude/settings.local.json to .gitignore
  • Remove from git history
S-03 Critical CSRF protection disabled via checkOrigin: false astro.config.mjs:12

Location: astro.config.mjs:12

Astro's built-in CSRF protection is explicitly disabled. All POST requests including Astro actions (/_actions/*) can be forged from any external origin.

Impact: Combined with missing admin authorization (S-04), an attacker can craft a page that silently escalates any authenticated visitor to admin role via a hidden form POST to /_actions/admin.updateUser.

Remediation:

  • Set checkOrigin: true (the Astro 5 default)
  • If specific routes need cross-origin, handle CORS per-route
