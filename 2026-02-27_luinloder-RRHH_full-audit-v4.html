01 / 06

Security

S-01 Critical Astro Actions bypass all authorization checks actions/cpo/cobertura.ts:15-201

Impact: The three Astro Actions (registrarCobertura, editarCobertura, eliminarCobertura) use supabaseAdmin (service_role, bypasses RLS) but never access context.locals.usuario to verify the caller's identity or role. Any authenticated user can create, edit, or delete any CPO professional's DDJJ records. No audit logging occurs.

// handler receives only input, no context — no auth check
export const registrarCobertura = defineAction({
  accept: 'form',
  input: ddjjRegistroSchema,
  handler: async (input) => {  // <-- no context parameter
    const { data, error } = await supabaseAdmin
      .from('ddjj_cpo').insert({...});

In contrast, the equivalent API route (/api/cpo/registrar.ts:33) correctly checks ownership.

Remediation:

  • Add the context parameter to all Action handlers and enforce role + ownership checks
  • Or deprecate these Actions entirely — the API routes already implement the same functionality with proper security
S-02 High Open redirect in auth callback pages/auth/callback.astro:16

Impact: The redirectTo query parameter is passed directly to Astro.redirect() without validation. An attacker can craft a URL like /auth/callback?code=VALID&redirectTo=https://evil.com to redirect authenticated users to a phishing site after login.

return Astro.redirect(Astro.url.searchParams.get('redirectTo') || '/');

Remediation: Validate that redirectTo is a relative path:

const raw = Astro.url.searchParams.get('redirectTo') || '/';
const redirectTo = (raw.startsWith('/') && !raw.startsWith('//')) ? raw : '/';
S-03 High Email-based auth_id auto-linking is unsafe middleware.ts:76-88

Impact: When no usuarios row matches by auth_id, the middleware falls back to email matching and permanently links the auth identity. If an admin pre-registers an email but the user hasn't logged in yet, an attacker controlling a different auth provider claiming that email could hijack the account.

if (!usuario && user.email) {
  const { data: byEmail } = await supabaseAdmin
    .from('usuarios').select(...)
    .eq('email', user.email)
    .is('auth_id', null).single();
  if (byEmail) {
    await supabaseAdmin.from('usuarios')
      .update({ auth_id: user.id }).eq('id', byEmail.id);
  }
}

Remediation: Add check: if (byEmail && user.email_confirmed_at) { ... }. Log auto-linking events to audit trail.

S-04 High shouldCreateUser: true enables auth.users pollution pages/auth/login.astro:390

Impact: The magic link OTP uses shouldCreateUser: true, creating new auth.users entries for any email. The client-side check-email call is trivially bypassable since the anon key is public. Combined with S-03, this enables account pre-hijacking.

const { error } = await supabase.auth.signInWithOtp({
  email,
  options: { emailRedirectTo: callbackUrl, shouldCreateUser: true }
});

Remediation: Set shouldCreateUser: false. The middleware at line 90 already blocks unregistered users, but auth.users pollution remains.

S-05 Medium No security response headers (CSP, X-Frame-Options, HSTS) middleware.ts, astro.config.mjs

Impact: Zero security headers are set. No Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, Strict-Transport-Security, or Referrer-Policy. The app is vulnerable to clickjacking, MIME sniffing, and protocol downgrade attacks. No vercel.json exists.

Remediation: Add headers via middleware or vercel.json: X-Frame-Options: DENY, X-Content-Type-Options: nosniff, Referrer-Policy: strict-origin-when-cross-origin, Permissions-Policy: camera=(), microphone=(), geolocation=().

S-06 Medium Known vulnerable dependency: xlsx 0.18.5 (SheetJS) package.json:27

Impact: SheetJS 0.18.5 has CVE-2024-22363 (prototype pollution via crafted spreadsheets). The package appears unused in source code (zero imports found).

Remediation: Remove with npm uninstall xlsx. If needed, migrate to exceljs.

S-07 Medium Stored XSS via innerHTML with user data pages/admin/usuarios.astro:358

Impact: User data (nombre, email, rol) is injected via innerHTML. If an admin stores malicious content in a user's name field, it executes as HTML when another admin views the user management page.

info.innerHTML = '<strong>' + data.nombre + '</strong> (' + data.email + ') — Rol: ' + data.rol;

Remediation: Use textContent instead of innerHTML.

S-08 Medium Universal supabaseAdmin usage bypasses RLS entirely lib/supabase.ts:17, all pages

Impact: Every database query uses supabaseAdmin (service_role key), completely bypassing Row Level Security. RLS provides zero defense-in-depth. If any middleware auth check has a bug, there is no secondary protection layer.

Remediation: Use user-scoped SSR client for read queries; reserve supabaseAdmin for admin-only writes. Implement RLS policies as defense-in-depth.

S-09 Medium supabaseAdmin silently falls back to anon key lib/supabase.ts:17

Impact: If SUPABASE_SERVICE_ROLE_KEY is not set, supabaseAdmin silently uses the anon key, causing unpredictable behavior.

export const supabaseAdmin = createClient(supabaseUrl, supabaseServiceKey || supabaseAnonKey);

Remediation: Throw an error at startup if the service role key is missing.

S-10 Low In-memory rate limiter resets on Vercel cold starts lib/rate-limit.ts:10

Impact: The Map-based rate limiter resets on each cold start. Only 3 routes are rate-limited; write endpoints are not.

Remediation: Use Vercel KV or Upstash Redis for persistent rate limiting. Add rate limits to write endpoints.

S-11 Low Email enumeration oracle via check-email pages/api/auth/check-email.ts:33

Impact: Returns { exists: true/false } allowing enumeration of registered emails.

Remediation: Always return a generic response regardless of whether the email exists.

S-12 Low Weak Zod enum validation on rol, rango, tipo_acceso lib/schemas/admin.ts:26, demanda.ts:55, informes.ts:25

Impact: Several schemas use z.string().min(1) instead of proper z.enum() for fields with known valid values, allowing arbitrary strings to pass validation.

Remediation: Replace with proper enum validation: z.enum(['admin', 'coordinador', ...]), z.enum(['08-20', '20-08', '24']), z.enum(['rol', 'usuario']).

02 / 06

Accessibility

A-01 Critical Login email input has no label pages/auth/login.astro:268

Impact: Screen readers cannot identify the purpose of the only input on the public-facing login page. Users rely on placeholder text which disappears on focus. WCAG 1.3.1, 4.1.2.

<input type="email" x-model="email" placeholder="tu@email.com"
       class="login-input" required autocomplete="email" />

Remediation: Add <label for="email">Correo electronico</label> or aria-label="Correo electronico".

A-02 Critical Modal component lacks focus trap components/shared/Modal.astro

Impact: When a modal is open, Tab focus escapes into the page behind it. This is confusing for screen reader users and violates the dialog pattern. The modal is used across the entire application. WCAG 2.4.3, 2.1.2.

Remediation: Implement a focus trap that cycles Tab between the first and last focusable elements inside the modal.

A-03 Critical Autocomplete dropdowns lack ARIA combobox pattern components/cpo/ResumenMesDemanda.astro, SolicitudesPendientes.astro, pages/cpo/demanda.astro

Impact: Custom autocomplete inputs built with Alpine.js lack role="combobox", aria-expanded, aria-controls, aria-activedescendant, role="listbox", role="option". Screen readers cannot navigate these. WCAG 4.1.2, 1.3.1.

Remediation: Implement the WAI-ARIA combobox pattern. The same autocomplete is duplicated 3 times — fix once and share.

A-04 Critical Chart.js canvas elements have no text alternatives 8 chart components across cpo/ and informes/

Impact: All <canvas> chart elements (8 components) have no text alternative. Screen readers announce nothing meaningful for complex data visualizations showing compliance rates, demand coverage, R/Q ratios, and trend data. WCAG 1.1.1.

<canvas id={chartId} data-json={dataJson}></canvas>

Remediation: Add role="img" and aria-label with a data summary to each canvas, or provide an adjacent <p class="sr-only"> description.

A-05 High Tables throughout app lack scope on <th> elements 30+ tables across all admin, CPO, and informes pages

Impact: Screen readers cannot associate data cells with their headers in ~30 tables. Also, most tables lack <caption> or aria-label to describe their purpose. WCAG 1.3.1.

Remediation: Add scope="col" to all column <th> elements. Add aria-label or <caption> to each table. Fix DataTable component once to cover many instances.

A-06 High Decorative SVGs lack aria-hidden="true" BaseLayout.astro, index.astro, all page navigation icons (50+ instances)

Impact: Inline <svg> icons throughout the app cause screen readers to attempt reading path data as garbled text. WCAG 1.1.1.

Remediation: Bulk add aria-hidden="true" to all decorative SVGs. For meaningful SVGs, use role="img" with aria-label.

A-07 High Form validation errors not announced to screen readers components/shared/Callout.astro, all form pages

Impact: Error callouts render as plain <div> without role="alert" or aria-live. Screen readers may not announce validation failures. WCAG 3.3.1, 3.3.3.

Remediation: Add role="alert" to Callout when type="error", and role="status" when type="success".

A-08 High UserMenu dropdown lacks keyboard navigation and ARIA components/layout/UserMenu.astro

Impact: No arrow key navigation, no role="menu", no aria-expanded, no aria-haspopup. WCAG 2.1.1, 4.1.2.

Remediation: Implement full ARIA menu pattern with keyboard support (arrows, Escape, Home/End).

A-09 High Mobile sidebar lacks Escape handler and focus trap components/layout/BaseLayout.astro

Impact: When mobile hamburger menu opens the sidebar overlay, no Escape key handler and no focus trapping. WCAG 2.1.1, 2.4.3.

Remediation: Add @keydown.escape to close; trap focus within when open.

A-10 High KPI cards with role="button" lack keyboard handlers components/cpo/PanelPrioridadPromocion.astro

Impact: Interactive KPI cards have role="button" and tabindex="0" with @click but no @keydown.enter/@keydown.space. Keyboard users cannot activate these. WCAG 2.1.1.

Remediation: Add @keydown.enter and @keydown.space.prevent handlers matching click behavior.

A-11 Medium Custom focus styles override global :focus-visible with low-contrast shadow 9+ page-level scoped style blocks

Impact: Many pages override the solid focus outline with box-shadow: 0 0 0 2px rgba(78, 205, 196, 0.1) (10% opacity), likely failing the 3:1 contrast requirement for focus indicators. WCAG 2.4.7.

Remediation: Remove outline: none overrides or increase shadow opacity to 0.4+.

A-12 Medium Multiple <h1> elements on several pages pages/index.astro:60-61, pages/informes/index.astro, pages/mi-residencia/*.astro

Impact: BaseLayout renders <h1> for page title, while some pages render additional <h1>s in content. The index page has three <h1> elements. WCAG 1.3.1.

Remediation: Use <h2> for page content headings when BaseLayout provides <h1>.

A-13 Medium Calendar toggle buttons lack accessible names and pressed state components/cpo/CalendarioDDJJ.astro, CalendarioDemanda.astro

Impact: Hundreds of toggle buttons in calendar grids have no aria-label with date/type context and no aria-pressed to reflect state. WCAG 4.1.2, 1.1.1.

A-14 Medium Progress bars lack role="progressbar" and ARIA attributes components/mi-residencia/ProgresoResidencia.astro, pages/mi-residencia/vacaciones.astro

Impact: Visual progress bars (residency progress, vacation usage) display percentages visually but have no role="progressbar", aria-valuenow, or aria-valuemax. WCAG 4.1.2.

A-15 Low Dynamic Alpine.js updates lack live region announcements residencias/simulador.astro, CalendarioDDJJ, CalendarioDemanda, ResumenMesDemanda

Impact: Interactive updates (simulator KPIs, calendar toggles, filter results) are not announced to screen readers. WCAG 4.1.3.

Remediation: Add aria-live="polite" regions for status updates.

P-01 Critical Middleware fires 2-3 Supabase queries on every authenticated page load middleware.ts:57-108

Impact: Every page navigation incurs: (1) getUser(), (2) profile lookup by auth_id, (3) fire-and-forget heartbeat UPDATE. On Vercel+Supabase, each round-trip adds 50-150ms. That's 100-300ms burned before page rendering even starts.

// Query 1: Validate session
const { data: { user } } = await supabase.auth.getUser();
// Query 2: Fetch profile
let { data: usuario } = await supabaseAdmin.from('usuarios').select(...)
  .eq('auth_id', user.id).single();
// Query 3: Heartbeat (fire-and-forget)
supabaseAdmin.from('usuarios').update({ ultima_actividad: ... }).then();

Remediation:

  • Cache user profile in JWT custom claims or short-lived cookie after first fetch
  • Throttle heartbeat to once per 60s (store timestamp in cookie)
  • Parallelize getUser() and profile lookup if possible
P-02 High xlsx package (547KB) is a dependency but never imported package.json:26

Impact: Bloats node_modules, slows installs and CI builds. May increase cold start time.

Remediation: Remove with npm uninstall xlsx.

P-03 High Chart.js imported with registerables — no tree-shaking lib/chart-loader.ts:1-11

Impact: Chart.register(...registerables) registers all chart types, scales, and plugins (~200KB). With selective imports, this drops to ~80KB.

Remediation: Import only needed controllers: BarController, BarElement, CategoryScale, LinearScale, Tooltip, Legend.

P-04 High Google Fonts: 3 families, 11 font files, render-blocking components/layout/BaseLayout.astro:47-49

Impact: Loading DM Serif Display (2 styles), DM Sans (7 weights), JetBrains Mono (2 weights) adds 200-400ms to FCP. The CSS request blocks rendering even with display=swap.

Remediation: Self-host fonts via @fontsource. Reduce DM Sans weights (300 and italic 400 appear unused). Use font-display: optional for JetBrains Mono.

P-05 High Pervasive SELECT * pattern across 45+ queries Multiple: admin/hospitales.astro:16, admin/usuarios.astro:16, cpo/registro.astro:73-94, admin/auditoria.astro:16

Impact: Over-fetches columns from every table. For audit_log with large JSON diff columns, this is especially heavy. For usuarios with .limit(500), all columns for 500 users.

Remediation: Replace every select('*') with explicit column lists. Prioritize audit_log and usuarios queries.

P-06 Medium Sequential queries on admin/index, snapshots-ddjj, and usuarios pages pages/admin/index.astro:51, pages/admin/snapshots-ddjj.astro:18-38, pages/admin/usuarios.astro:16-54

Impact: Independent queries run sequentially instead of being combined into existing Promise.all() blocks, wasting 100-300ms total across these pages.

Remediation: Move all independent queries into a single Promise.all().

P-07 Medium No Supabase response caching anywhere All page files

Impact: Frequently-read, rarely-written data (hospitals, rules, professional lists) is re-queried on every page load by every user.

Remediation: Implement simple in-memory cache with TTL for static-like data. Consider Vercel ISR for non-real-time pages.

P-08 Medium Duplicate CSS in scoped <style> blocks across pages Every .astro page (shared patterns: .filtros-bar, .admin-section, .footer-links, .kpi-grid)

Impact: Astro scopes styles per-component, so each page ships its own copy of identical rules. Estimated 3-5KB of duplicated CSS per page.

Remediation: Extract shared patterns into globals.css or shared Astro components.

P-09 Low CPO dashboard embeds large inline JSON payload for charts pages/cpo/dashboard.astro:373

Impact: The entire resumenParaGrafico array (potentially 50+ professionals x 12 months) is serialized and embedded in HTML. Could be 50-100KB.

Remediation: Serve chart data via an API route and fetch client-side.

SEO

E-01 High No robots.txt — internal app exposed to crawlers public/ (file missing)

Impact: Without a robots.txt, search engines attempt to crawl all URLs. For a login-gated HR system handling sensitive employee data (DNI, medical records), this is a privacy concern. URLs like /admin/usuarios and /cpo/registro leak app structure.

Remediation: Create public/robots.txt: User-agent: * / Disallow: / / Allow: /auth/login / Allow: /legal/

E-02 High No <meta name="robots"> tag on any page BaseLayout.astro, login.astro, privacidad.astro

Impact: Even with robots.txt, pages discovered via external links may be indexed. A noindex, nofollow tag is a stronger signal for authenticated pages.

Remediation: Add <meta name="robots" content="noindex, nofollow" /> to BaseLayout <head>.

E-03 Medium Login page missing <meta description> pages/auth/login.astro:18-29

Impact: The primary public page has no meta description. Search engines auto-generate a snippet from garbled UI text.

Remediation: Add <meta name="description" content="Acceso al sistema RRHH de Anestesiologia AMA Mendoza." />

E-04 Medium No OG / Twitter Card meta tags anywhere All page templates

Impact: When the login URL is shared on Slack, WhatsApp, or any messenger, no preview card is shown — just a bare URL.

Remediation: Add og:title, og:description, og:image, twitter:card to the login page at minimum.

E-05 Low Privacy page missing Spanish accents throughout pages/legal/privacidad.astro

Impact: Legal text uses ASCII-only Spanish ("Politica", "Retencion", "autenticacion") which looks unprofessional.

Remediation: Add proper accents: "Politica" → "Politica", "autenticacion" → "autenticacion", etc.

E-06 Low Web manifest missing start_url public/site.webmanifest

Impact: Without start_url, PWA install prompt uses the current URL as entry point.

Remediation: Add "start_url": "/".

F-01 High guardar-batch.ts missing CPO ownership check pages/api/cpo/guardar-batch.ts:15-17

Impact: Unlike registrar.ts, registrar-bulk.ts, and eliminar-todo.ts, the batch save endpoint does not verify that a CPO user's profesional_cpo_id matches the submitted profesional_id. A CPO user could overwrite another professional's entire month of DDJJ records (the endpoint does full delete-then-insert).

// guardar-batch.ts — NO ownership check:
const usuario = context.locals.usuario;
if (!usuario) { return redirect({ error: 'No autenticado' }); }
// Proceeds to delete + insert for ANY profesional_id

// Compare: registrar-bulk.ts HAS the check:
if (tieneRol(usuario, 'cpo') && !tieneRol(usuario, 'admin')
    && usuario.profesional_cpo_id !== profesional_id) {
  return context.redirect(...);

Remediation: Add the same ownership check present in all other CPO write endpoints.

F-02 High Non-atomic delete-then-insert in guardar-batch.ts pages/api/cpo/guardar-batch.ts:52-88

Impact: The batch save deletes all existing records for the month, then inserts new ones as two separate database operations. If the INSERT fails (network error, constraint violation), the DELETE has already succeeded and the professional's data for that month is lost.

// Step 1: Delete existing — succeeds
const { error: deleteError } = await supabase
  .from('ddjj_cpo').delete()
  .eq('profesional_id', profesional_id)
  .gte('fecha_cobertura', primerDia);

// Step 2: Insert new — could fail, leaving month empty
const { error: insertError } = await supabase
  .from('ddjj_cpo').insert(rows);

Remediation: Use a Supabase RPC function (database transaction) to wrap as atomic. Alternatively, add rollback from snapshot if insert fails.

F-03 Medium eliminar.ts ownership check compares form values against each other pages/api/cpo/eliminar.ts:8-56

Impact: The delete endpoint checks cobertura.profesional_id !== profesional_id, but both values come from user-supplied form data. A CPO user could submit matching ddjj_id and profesional_id of another professional, passing the "ownership" check.

Remediation: Compare cobertura.profesional_id against usuario.profesional_cpo_id from the session.

F-04 Medium Timezone inconsistency: demanda uses UTC, CPO uses Argentina time lib/demanda-utils.ts:13 vs lib/cpo-utils.ts:46

Impact: validarVentanaDemanda uses new Date() (UTC on server) while validarVentanaAbierta uses ahoraEnArgentina(). Since Argentina is UTC-3, the DDJJ window can open/close 3 hours earlier for demanda than for CPO.

// demanda-utils.ts — raw UTC:
const hoy = new Date();
// cpo-utils.ts — timezone-aware:
const hoy = ahoraEnArgentina();

Remediation: Replace new Date() with ahoraEnArgentina() in demanda-utils.ts.

F-05 Medium listUsers() fetches all auth users without pagination pages/api/admin/usuarios/crear.ts:64

Impact: Fetches ALL auth.users entries into memory to find a single email match. Does not scale.

const { data: { users: authMatches } } = await supabase.auth.admin.listUsers();
const authMatch = authMatches?.find(u => u.email?.toLowerCase() === email);

Remediation: Use supabase.auth.admin.getUserByEmail() or filter server-side.

F-06 Low TOCTOU race condition in CPO duplicate check pages/api/cpo/registrar.ts:43-54

Impact: Duplicate check (SELECT) and INSERT are separate operations. Concurrent requests could both pass and create duplicates.

Remediation: Add a unique constraint on (profesional_id, hospital_id, tipo_cobertura_id, fecha_cobertura).

F-07 Low Non-atomic delete-then-insert in acceso.ts and guardar.ts (demanda) pages/api/informes/acceso.ts:48-67, pages/api/demanda/guardar.ts:67-88

Impact: Same pattern as F-02 but lower severity — access rules and demanda details are less critical than DDJJ data.

Remediation: Wrap in database transactions for robustness.
